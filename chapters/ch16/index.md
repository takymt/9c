# 複数の型をまとめる

- 複数の型をまとめるには `struct` 型
  - 構造体タグ: `struct` 型の型名
  - 構造体変数: `struct` 型で宣言された値
  - 関数よりも先に宣言する
  - 変数名とは別の名前空間
  - structの`{}`はスコープ定義ではなく宣言であるため、末尾には`;`が必要
- 構造体のメリット
  - 関連するデータをまとめられる
  - 可読的
  - 関数や配列にわたすときや代入時にsimple
- 構造体の代入は再帰的に処理される
  - ただしポインタはそのままポインタとして扱われる
- 既存の型にエイリアスを貼る`typedef`
  - `typedef struct point {...} point` で `struct point`という形に`point`というエイリアスが貼られる
    - `point`: 構造体タグ
    - `struct point`: 構造体型
    - `point`: 構造体型 = `struct point`
  - この場合、構造体タグと構造体型の名前空間は異なるが、構造体型と変数の名前空間は同じ
    - `point point = {...}` はできない
    - `struct point point = {...}` はできる
  - 構造体タグと構造体型を宣言することができ、構造体型を宣言する場合は構造体タグを省略できる
  - 派閥として`typedef`を使わないことも多い
- 命名規則
  - 構造体タグ: 小文字 + snakecase
  - 構造体型:
    - パターン1: 小文字　+ snakecase + `_t`
    - パターン2: PascalCase
    - パターン3: `typedef`自体使わない
- 構造体の定義とメンバアクセス
  - `{ .id = 1, ... }` のように初期化
  - 値の場合は`student.id`、ポインタの場合は`(*student).id`または`student->id`
