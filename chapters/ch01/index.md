# 1章：世界最小のプログラム

- 何もしないプログラム
  - C言語のプログラムは関数が第一級オブジェクトではない
  - C言語の関数に順番はない
  - main関数は必須であり、1番初めに動作する
- コンパイラは翻訳ソフト
  - 2進数 -> 16進数 -> アセンブラ -> フォートラン(世界初プログラミング言語) -> C言語
  - 人間の言葉に近い言語(高級言語)を機械語に翻訳する
    - 一気にまとめて翻訳: コンパイラ
    - 同時翻訳: インタプリタ(デバッグ製は高いが速度が犠牲)
  - ビルド(広義のコンパイル)の流れ
    - プリプロセッサ: 文字列の調整(空白や改行の結合、記号の置き換え、#define疑似命令など)
    - コンパイル: より高速なマシン語に変換
    - リンク: コンパイルされたファイル同士を結合

## 問題回答

- 関数
- `<type> <funcName>(<args>) {<process>}`
- コンパイラ
- [do_nothing.c](./do_nothing.c)
- C言語の関数には順番の概念がないが、それだとどの関数を最小に実行すべきかわからない。最初に実行する関数としてのmainが必要。
