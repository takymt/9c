# ファイルの取り扱い

- ファイル操作: ファイルを開く`fopen` -> ファイルを操作する -> ファイルを閉じる`fclose`
- `fopen` のモード
  - r/w/a: 読み込み/新規書き込み/追記
  - r+/w+/a+: 読み書き
  - b: バイナリ
- FILE型を使用する
  - FILEへのポインタ: ファイルポインタ
- `fclose` の役割: `fopen`によるロックの解除
- リトルエンディアンとビッグエンディアン
  - 100が`6400`になるのがリトル、`0064`になるのがビッグ
  - リトルは2桁ごとに逆順
  - リトルのほうが意味がある数字が先頭に来やすいので利点がある
- 文字列の配列: `char *argv[]`
  - `配列はポインタ` -> 文字列 = charへのポインタに等しい
  - 文字列の配列 = 最初の文字列へのポインタ = 最初の文字へのポインタへのポインタ
- 配列はポインタに格納された時点でポインタだが、定義場所と同じスコープ内ではまだ配列(実態を持つ)
  - `size_t len = sizeof(arr) / sizeof(arr[0]);` が出来るのは同じスコープのみ
  - 基本的には関数引数で`size_t len`を渡す
