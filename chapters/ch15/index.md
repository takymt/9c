# ポインタ変数の仕組み

- 32bitコンピュータ = 32bitごとにメモリを読み取る
- メモリのフォーマット演算子は`%p` (pointer)
- arrayは0要素目へのポインタ, index分のアドレスを加算している
- 値渡しは関数に単なる数値を渡すのに対して、参照渡しはアドレスを渡す
  - これにより変数自体を直接書き換えることが出来る
- ポインタ: アドレスを記憶する
  - ポインタ型
    - ポインタ変数であることを表す型
    - 他の型から作られる派生型
    - 元の型とポインタ型を含めて記録しておくことで、取り出した際に元の型に基づいてメモリを解釈できる(intであればここから32bitみたいな)
    - 元の型を必要としない`void`というポインタ型があるが、これは元の値の形がわからないため値を取り出せない
    - ポインタへのポインタ型も存在可能
  - ポインタ値
    - アドレス値そのもの
    - ポインタは符号なし32bit
    - intとは意味的に違うので分けている
  - ポインタ変数
    - ポインタ値を記憶する変数
    - 記憶しているアドレスのメモリを読み書きできる
    - ポインタ値(p)としてと、ポインタ先の変数(*p)として両方振る舞える
    - ポインタ変数の宣言方法は2種類
      - `int* p`: 型としてわかりやすいが、`int* p1, p2`だとp2はただのint
      - `int *p`: 型としてわかりにくいが、ポインタであることは明白
- ポインタに代入するアドレスはOSの仮想メモリに管理されている
  - 連続しているように見せる
  - メモリ不足でもディスクを使って大きなメモリがあるかのように振る舞う
  - プログラムごとのメモリを隔離する
- 適当なアドレスを代入するとOSの仮想メモリに反するので強制終了する
  - 正常なアドレスを入れるには通常の変数のアドレスを`&`演算子で取得する
- NULLポインタ
  - ポインタ変数の宣言直後はデタラメな値が入っている
  - その値が有効なアドレスかはわからないので明示的に`int *p = NULL`としておく
  - `if (p == NULL)` みたいなことができる
  - 文法上 `int *p = 0` としても NULL ポインタ相当である
  - 仮想アドレスの0番地は使わないのでNULLが0bitなのと相性がいい
- `*`演算子には、乗算/参照/ポインタ型定義の3つの意味があり、それぞれ別々
- 配列を引数で渡す場合は、配列の最初のメモリアドレスを渡しているのに等しい
  - 以下は全て同じ、2番目がベスト
    - `int get_average(int data[10])`: 書く必要がないサイズを書いていて幼稚
    - `int get_average(int data[])`: 配列であることがわかりサイズとも無関係
    - `int get_average(int *data)`: 配列ではないポインタと間違えやすい
- 配列宣言時の`[]`と、配列読み書き時の`[]`は全く別の演算子
  - 前者: 配列のサイズを定義
  - 後者: 配列(=メモリアドレス)にいくつアドレスを足すかという加算の演算子
  - `data[i]`は`*(data + i)`と同じであり、dataは`&data[0]`と同じ
- ポインタ専用の書き方として `*(ポインタ + 要素番号)`
- `int i`ではなくポインタ変数をループ内でインクリメントすることで配列へのアクセスが高速化されるが、現在のコンパイラはその程度の工夫は自動で行ってくれる
  - 配列アクセスは++ポインタ演算も多いが、基本は`[]`アクセスがよみやすくて良い
- ポインタ = 変数を操作するためのショートカット
  - `&`でショートカットとして保持し、`*`で通常変数モードとして操作する
  - ポインタやアドレス周りでバグが発生した際は、ポインタ=アドレス各納であることを思い出す
